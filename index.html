How does Javascript work?

Javascript is a client side scripting language. Client side scripting language means browser runs these scripts and there are two trees in the browser, one is the dom tree and the other is the style structure, these two are called render trees again. Chrome browser has its own engine called v8 to run javascript code. Javascript is an interpreted language that does not need to be compiled. Javascript code that is executed with the editor is made with c ++ language. Javascript engine first parser the code to create the tree then interpret. JavaScript code is run in an execution context. An execution context is created when running a JavaScript program. JavaScript code contains specific memory allocation for variables and functions.

Different between local storage and session storage?

Local storage is paramanent on the other hand session storage is not paramanent. If data is stored in local storage afterwords colse the browser data do not remove or delete but if data is stored in session storage then colse browser data will be romoved. These are the main different between local storage and session storage. Local storage stores data with no expiration date but session storage data is cleaed when tab is colsed. Another difference is session storage is accessed with the sessionStorage variable and local storage uses the localStorage variable. If we want to remove data from local storage then we remove data from local storage manually if we removedata from session storage then we close tab or page then automatically remove data from session storage. LocalStorage and sessionStorage both are called WebStorages.
